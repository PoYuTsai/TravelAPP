# Phase 4：行程案例展示系統

> 日期：2026-01-23
> 狀態：📋 設計完成，待實作

---

## 概述

Phase 4 專注於取代 Rezio，在自己的網站建立行程展示功能，展示招牌套餐 + 過去案例，導向 LINE 諮詢。

| Phase | 性質 | 目的 | 狀態 |
|-------|------|------|------|
| Phase 1 | 公開網站 | 轉換率優化 | ✅ 完成 |
| Phase 2 | SEO 優化 | 流量獲取 | ✅ 完成 |
| Phase 3 | 內部工具 | 營運效率 | ✅ 完成 |
| **Phase 4** | **行程展示** | **取代 Rezio** | 📋 設計完成 |

### 為什麼要取代 Rezio？

| Rezio 功能 | 實際使用狀況 |
|-----------|-------------|
| 金流刷卡 | ❌ 一年只有 2 人用 |
| 預訂系統 | ❌ 客人都走 LINE |
| **展示功能** | ✅ 這是唯一在用的 |

**結論：付訂閱費買的是「展示櫥窗」，這可以自己做。**

### 品牌定位考量

| | 大型旅行社 | 清微旅行 |
|---|---------|---------|
| 策略 | 多產品、標準套裝 | 客製化、親子專業 |
| 競爭方式 | 價格戰、規模 | 信任、專屬感 |
| 套餐定位 | 可直接購買的產品 | 展示用的範例行程 |

**套餐 = 行銷工具，不是產品**
- 讓客人快速理解「你們能做什麼」
- 降低「我該怎麼問」的門檻
- 最終導向 LINE 客製報價

## 4.4 技術架構

### 資料來源分工

```
┌─────────────────────────────────────────────────┐
│                    /tours 頁面                   │
├──────────────────────┬──────────────────────────┤
│   招牌套餐 (2-3個)    │    過去案例 (114個)       │
│                      │                          │
│   來源：Sanity CMS   │    來源：Notion API      │
│   維護：手動編輯      │    維護：自動同步         │
│   更新頻率：偶爾      │    更新頻率：即時         │
└──────────────────────┴──────────────────────────┘
```

**核心原則：不要重複輸入資料**

| 內容 | 存在哪 | 為什麼 |
|------|--------|--------|
| 過去案例（114個） | Notion | 已經有了，不搬 |
| 招牌套餐（2-3個） | Sanity | 需要精緻設計，手動控制 |

### Sanity Schema: tourPackage

```typescript
// 欄位設計

// 基本資訊
title          // 標題（親子經典清邁）
slug           // 網址（family-classic-5d）
subtitle       // 副標（適合第一次來清邁的親子家庭）
coverImage     // 封面圖（精美設計圖）
duration       // 天數（5天4夜）
highlights[]   // 亮點標籤（大象營、夜間動物園...）
order          // 排序

// 行程內容
overview       // 行程總覽（純文字或 Rich Text）
dailySchedule[] // 每日行程
  - day        // 第幾天
  - title      // 標題（抵達清邁・輕鬆適應）
  - activities // 活動內容
includes[]     // 包含項目（中文導遊、兒童座椅...）
excludes[]     // 不包含項目

// 價格資訊
priceNote      // 價格說明（「NT$3,200/天起，依人數調整」）
cta            // 行動呼籲（LINE 諮詢客製報價）
```

### Notion API：過去案例

複用現有的 Dashboard API，只取需要的欄位：

```
從 Notion 拉取：
├── 客戶名稱    →  顯示名稱
├── 旅遊日期    →  月份 + 判斷狀態
├── 天數       →  從日期範圍計算
└── 行程框架    →  截取前 50 字當摘要（可選）

不拉取（隱私）：
├── 總收入
├── 總成本
├── 利潤
└── 支付狀態
```

## 4.5 AI 工具協作流程

### 製作招牌套餐的流程

```
Step 1: 內容發想（你 + AI）
        ↓
Step 2: 文案撰寫（Claude/ChatGPT）
        ↓
Step 3: 視覺設計（Canva/Gamma）
        ↓
Step 4: 上傳到 Sanity
        ↓
Step 5: 完成上線
```

### Step 1: 內容發想

提供：
- 這個套餐要去哪些地方？
- 目標客群是誰？
- 跟其他行程有什麼不同？
- 最大的賣點是什麼？

### Step 2: 文案撰寫（AI）

| 工具 | 用途 |
|------|------|
| Claude | 寫行程描述、賣點 |
| ChatGPT | 寫標語、副標題 |

### Step 3: 視覺設計

| 工具 | 用途 |
|------|------|
| **Canva** | 封面圖、行程圖卡（主力工具） |
| Gamma.app | 快速做精美簡報式頁面 |
| Midjourney | 生成氛圍圖（沒照片時） |

**封面圖尺寸：1200 x 630 px**

### 時間估算

| 項目 | 時間 |
|------|------|
| 內容發想 + AI 文案 | 30 分鐘 |
| Canva 設計封面 | 30 分鐘 |
| 上傳 Sanity + 調整 | 15 分鐘 |
| **一個套餐總計** | **約 1.5 小時** |
