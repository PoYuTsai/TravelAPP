## 2. 測試順序（Tier A：stub，零真 API）

> 全程 `responder=stub`。Bot「回」＝回一段 stub 字串，目的是驗**路徑與閘**，不是驗內容品質。內容品質（masked 安全版）留到 §B（需授權）。
> 每一步：Eric 在私測群操作 → 觀察 → 對照 pass/fail 標準 → 點頭才進下一步。

### Step A1 — 未 tag / 未 quote → bot 不回
- **操作**：在私測群發一則一般訊息，**不** @bot、**不** quote bot 先前訊息。
- **預期**：bot 完全不回。
- **pass**：群內無任何 bot 回覆。
- **fail**：bot 回了 → 立即 §4 回滾，回報。
- **機制**：`botDirected = mentionsBot || isBotAuthoredQuote`；兩者皆 false → `shouldIgnoreCasualPartnerGroupChat` 忽略。

### Step A2 — tag bot 簡單問候 → bot 回
- **操作**：在私測群 @bot（或用 wake word `清微AI` / `清微助理`）發「你好」之類問候。
- **預期**：bot 回一段 stub 回覆。
- **pass**：bot 回 1 則、且只回 1 則（無重複）。
- **fail**：不回 / 回多則 → 回報（不回＝mention 偵測或 route 問題；多則＝去重 `claimPartnerReply` 問題）。
- **機制**：`botDirected=true` + decision worthy + 活 replyToken → send gate 4 條全真 → reply。

### Step A3 — tag bot 問「客需整理」 → bot 回（Tier A 為 stub 字串）
- **操作**：@bot 發一則「幫我整理這個客人需求 …」。
- **預期（Tier A）**：bot 回 stub 字串（**不是**真 masked 整理；真整理需 §B 授權）。
- **pass**：有回、單則、無洩漏（stub 不含任何 PII/景點名/金額）。
- **fail**：回多則 / 內容異常 → 回報。
- **註**：真 masked/安全版整理（base responder=anthropic 或 RAG draft）屬 §B，Tier A 不驗內容。

### Step A4 — customer OA 訊息 → 不自動回客人
- **操作**：用另一個（非測試群）LINE 帳號，以**一對一 OA**（私訊官方帳號）發訊息給 bot。
- **預期**：客人**不會**收到任何自動回覆；訊息僅被收為 case。
- **pass**：客人端零自動回覆。
- **fail**：客人收到回覆 → **最高優先**，立即 §4 回滾並回報（理論上結構性不可能）。
- **機制**：OA plane `botDirected` 永遠 false（`webhook-runtime` 寫死）；`canAutoReplyToOaCustomer` 對 `line_oa` 永遠 denied。

### Step A5 — send intent 不明確 → 不送
- **操作**：此步驗 operator 端「草稿 vs 送出」語意。情境＝有一個 routing decision 落在 `draft`（無 `sendTarget`），而非 `post_to_partner_group`。
- **預期**：bot **不**把該草稿貼進群。
- **pass**：群內無該草稿被貼出。
- **fail**：草稿被自動貼出 → 回報（B4 `canPostToPartnerGroupFromDC` 應要求 `sendTarget` 才送）。
- **機制**：B4 權限要求 `command.sendTarget.channel === 'line_partner_group'`；缺 `sendTarget` → draft only，不送。

## 3. 每一步 pass / fail 總表

| Step | 驗什麼 | pass | fail（任一即停） |
|------|--------|------|------------------|
| A1 | 未 tag 不回 | 群內無 bot 回覆 | bot 回了 |
| A2 | tag → 回 1 則 | 單則回覆、無重複 | 不回 / 多則 |
| A3 | 問客需整理 → 回（stub） | 單則、零洩漏 | 多則 / 異常內容 |
| A4 | customer OA 不自動回 | 客人端零回覆 | 客人收到回覆（最高優先） |
| A5 | 無 send intent 不送 | 草稿未貼群 | 草稿被自動貼出 |

**Tier A 整輪 pass 條件**：A1–A5 全 pass，且 A4 經 Eric 親測確認客人零回覆。任一 fail → §4 回滾 + 回報，**不**進 §B。

## 4. 一鍵回滾

| 動作 | 指令 / 操作 | 效果 |
|------|-------------|------|
| **主回滾（一個 env）** | `LINE_PARTNER_GROUP_ID` 改為占位值（必填 env，**不可清空**；或把 bot 踢出群） | normalizer 比對不符 → 事件不標 `line_partner_group`，partner reply 路徑整條停，不送 LINE |
| 確認回滾 | 重啟 / 重新部署後，於私測群 @bot → 應**完全不回** | 驗 partner 路徑確實停 |
| 退 responder（若曾翻 §B） | `AI_AGENT_PARTNER_RESPONDER_MODE=stub` | 退回零 billed |
| 退 RAG（若曾翻） | 移除 / 設非 `true`：`AI_AGENT_NOTION_RAG_ENABLED`、`AI_AGENT_PARTNER_RAG_DRAFT_ENABLED` | RAG draft 不浮現 |

> 回滾優先序：先改 `LINE_PARTNER_GROUP_ID`（結構閘，最徹底；改後需重部署生效），再依需要退 responder / RAG。（⚠️ 修正：原寫 `LINE_ROUTE_MODE=discord_smoke`，該 env 無消費者，翻它**無效**。）

## 5. 禁止事項（再次明列，對應 §0）

1. **不接正式夥伴群** — `LINE_PARTNER_GROUP_ID` 全程只指私人測試群。
2. **不回客人** — customer OA auto-reply 結構性 OFF，私測只驗「確實不回」。
3. **不寫 Sanity** — quote 全程 dry-run，無 write token。
4. **不產正式 quote URL** — 只 `DRAFT-<caseId>` / `isOfficial:false`。
5. **不洩漏 Notion / private data** — RAG 雙閘維持關；即使 §B 也只驗 masked。
6. **不跑真 API（除非 Eric 明確授權）** — Tier A 全 stub；§B 需 Eric 逐次點頭。
7. **CC 不自動翻 gate、不自動 push、不代改 `.env.local`、不真送 LINE。**
