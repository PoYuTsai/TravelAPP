## 決策（Eric 2026-06-09）：A 案 — live retrieval 只做 theme signal，不代入 draft

Notion live 的 operator-safe summary（`OperatorSafeCaseSummary`）為避免 PII，**結構上不含景點名**
（`itinerarySnippet` 整段因 GAP-1 丟棄），只有 `areaHints`/`themeHints` + 天數/人數/車型。因此 live
retrieval **無可靠景點名**，只能貢獻主題訊號；具體名稱代入 draft 留待後續 curated whitelist。

### Guardrails（Eric 補充，pinned by tests）

1. **provenance 預設保守**：舊 fixture 不填 `provenance` 視為 `fixture`（可代入）；但 live mapper 產出的
   **每一筆強制 `provenance:'live_masked'`**，不允許 caller override 成 fixture。
2. **name 只能來自 `GENERIC_THEME_LABELS`**：不可從 Notion snippet/title/summary 抽字串當 name。
   即使輸入含具體景點名，輸出 name 仍只能是 generic label。
3. **substitution guard 放在 `pickRetrievalAlternative` 最核心處**：不是只靠 mapper 不給資料；即使有人
   手動塞 `live_masked` + `name`，也**不得** substituted。
4. **CLI dry-run 輸出偏 masked summary**：可顯示 theme / count / outcome / operator preview；不印 raw
   Notion result / URL / DB id / 客戶姓名 / 成本 / 收入 / 利潤 / 具體景點名。

## Cut 1 — 純 TS 核心（vitest 權威閘，leak-critical）

### `RetrievalCaseRef.provenance`
`customer-itinerary-change-composer.ts`：加 optional `provenance?: 'fixture' | 'live_masked'`。
- 不填 ⇒ 等同 `fixture`（向後相容，既有 fixtures 不動）。

### substitution guard（核心）
`pickRetrievalAlternative`：substitution 分支的候選必須 `provenance !== 'live_masked'`。
- 即使手動塞 `{ name:'茵他儂', themeTag:'nature', mobilityFriendly:true, provenance:'live_masked' }`
  且 request themeTag 對齊，outcome 仍只能 `named_only`（候選仍可被建議），**永不 substituted**。
- `named_only` 候選清單仍含 live_masked（可供人工建議），只是不代入。

### `live-masked-retrieval-cases.ts`（新檔）
- `MOBILITY_FRIENDLY_THEMES`：code-only 保守白名單（`cafe` / `massage` / `photo` / `shopping` /
  `market` / `temple` / `family`；排除 `zipline` / `adventure` / `elephant` / `zoo` / `night_safari`）。
- `GENERIC_THEME_LABELS`：theme token → 泛稱中文 label（如 `cafe`→「較輕鬆的咖啡主題景點」），絕無
  具體景點名/數字。
- `toLiveMaskedRetrievalCases(summaries: OperatorSafeCaseSummary[]): RetrievalCaseRef[]`：
  - 每筆 summary 取**第一個落在白名單**的 themeHint；無則該筆不產出（→ none）。
  - 強制輸出 `{ name: GENERIC_THEME_LABELS[theme], themeTag: theme, mobilityFriendly: true,
    provenance: 'live_masked' }`，依 themeTag 去重（第一筆 wins，穩定）。
  - name 一律取自 `GENERIC_THEME_LABELS`；summary 即使夾帶具體名（理論上不會，因 masked）也不影響輸出。

### preview conditional
`customer-change-operator-preview.ts`：`named_only` 分支若 candidates 含 `live_masked`，`來源 case` 改顯示
「Notion live（masked，僅 theme=<t> 訊號，無景點名，依政策不代入）」。fixture 路徑不變（不破 M3.3e snapshot）。

### Cut 1 契約（tests）
1. mapper：themeHints=['cafe'] → 一筆 live_masked，name=generic，mobilityFriendly true，provenance live_masked。
2. mapper：themeHints=['zipline','adventure']（無白名單 theme）→ 不產出。
3. mapper：themeHints=['zipline','cafe'] → 取 cafe（跳過不適合，取第一個白名單）。
4. mapper：多筆同 theme → 去重一筆。
5. mapper：輸入夾帶具體景點字串 → name 仍只能是 generic label（鎖 guardrail 2）。
6. mapper：每筆強制 provenance live_masked，caller 不能 override（鎖 guardrail 1）。
7. guard：手動塞 live_masked + name + 對齊 themeTag → composeCustomerChange outcome named_only（非 substituted），
   draft 不含該 name（鎖 guardrail 3）。
8. mapper→compose：live cases 餵 addActivities 同 theme → named_only，draft 無 generic name。
9. preview：live_masked named_only 顯示 masked-source 行，無具體景點名。
