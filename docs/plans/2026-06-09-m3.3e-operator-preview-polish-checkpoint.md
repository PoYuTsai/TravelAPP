## 1. operator preview polish

`buildOperatorRetrievalPreview(applications) → string[]`：每筆 declined activity 一個**標籤化區塊**
（單一多行字串），固定欄位讓 Eric 一眼判斷：

- `原需求`（declinedActivity）
- `否決原因`（新增 `RetrievalApplication.declineReason` = 觸發否決的 unsuitable token，例「長輩不適合（叢林飛索）」）
- `是否代入 draft`（substituted=是／named_only=否（僅建議）／none=否）
- `替代候選`（候選名 + 共 N 筆，皆來自 retrieval 白名單；無則「無」）
- `來源 case`（substituted 顯示代入來源 + theme；none 顯示「請人工補景點，系統不杜撰」）

實作：`customer-itinerary-change-composer.ts` 的 `RetrievalApplication` 加 `declineReason?`，
`applyChanges` 在 decline 分支 `app.declineReason = hit`。preview 為內部用，theme/來源/候選數可顯示。
