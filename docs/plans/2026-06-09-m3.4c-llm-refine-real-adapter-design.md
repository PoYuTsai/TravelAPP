# M3.4c — LLM Refine Real Adapter + Offline Smoke（設計）

- 日期：2026-06-09
- Branch：`codex/line-oa-agent-mvp`
- 前置：M3.4b（LLM refine offline harness：`RefineDraftSource` seam + 三道 deterministic guard + fail-closed）已綠並 push（tip `2df03d6`）。

## 1. 目標與邊界

把**真 LLM adapter** 接進 M3.4b 的 `RefineDraftSource` seam，做 **operator-CLI-only 的 offline smoke**：跑同一批 fixture 草稿，過同一條 `refineCustomerItineraryDraft` 三道 guard，統計 accepted / rejected / fallback rate。

本刀**只能**：
- 在 seam 後 slot-in 真 adapter。**不動** `refineCustomerItineraryDraft` harness 與三道 guard（lint / structuralDiff / leak）一行。
- adapter 為 lib 純工廠 + 可注入 `callModel`，CLI 為薄的 gated assembler（仿 `notion-rag-dry-runner.mjs`）。
- 真 LLM output 必過 M3.4b 三道 guard；任一 fail → fallback deterministic draft。

本刀**不**：不接 LINE、不翻 customer/partner gate、不寫 Sanity、不產正式 quote URL、不讓 LLM 繞過 deterministic/lint gate。

硬邊界（禁止印出）：API key、token、DB id、Notion URL、成本、收入、利潤、客戶姓名、draft 全文、prompt 全文、internal/operator data。

## 2. 五項修正（本刀明確納入）

1. **gate reason 細分**：loader 不只回 `skipped/client_not_wired/real` 三大類，而是細到可區分根因（見 §4）。
2. **prompt forbidden terms 獨立 reason**：input-leak 不再併入 `source_error`，獨立為 `prompt_leak`（見 §3.3、§5）。
3. **prompt builder 嚴禁結構化內部欄位**：`buildRefinePrompt` 簽章**只**收 `deterministicDraft: string`，結構上無法夾帶 `operatorNotes` / `retrievalApplications` / `provenance` / `themeTag` / `constraints`；並以測試鎖死這些 token 不得出現在組好的 prompt（見 §3.2、§6）。
4. **real smoke 不自動跑**：任何真打 Anthropic API 的 smoke 一律等 Eric 明確指令手動執行；CC/tmux 不代跑（見 §7）。
5. **model default 集中管理、可配置**：單一常數 `REFINE_MODEL_DEFAULT = 'claude-haiku-4-5'`，可由參數或 env `AI_AGENT_REFINE_LLM_MODEL` 覆寫（見 §3.4）。

## 3. lib adapter：`src/lib/line-agent/notion/llm-refine-adapter.ts`

純 + 可注入；唯一非純處是預設 `callModel` 動態 import `@anthropic-ai/sdk`。

### 3.1 model 常數（修正 5）

```ts
export const REFINE_MODEL_DEFAULT = 'claude-haiku-4-5'
/** 解析順序：顯式 model 參數 > env AI_AGENT_REFINE_LLM_MODEL > REFINE_MODEL_DEFAULT。 */
export function resolveRefineModel(opts?: { model?: string; env?: Record<string, string | undefined> }): string
```

### 3.2 prompt builder（修正 3）

```ts
export interface RefinePrompt { system: string; user: string }
/** 故意只收草稿字串：結構上不可能夾帶 operatorNotes / retrievalApplications /
 *  provenance / themeTag / constraints 等內部欄位。 */
export function buildRefinePrompt(deterministicDraft: string): RefinePrompt
```

- `system`：freeze 指令——只暖化第一個 Day 之前的開場白與標題行措辭；逐字凍結日期行/人數行/Day 數與順序/每日 dateLabel/每個活動行/午晚餐/住宿；最後一天早上送機不得出現餐宿；只回完整草稿、不要前後綴、不要 code fence、不要說明。
- `user`：**只**放 `deterministicDraft`。
- output 不做正規化（避免遮蔽 drift），原樣交給三道 guard。

### 3.3 input forbidden-terms tripwire（修正 2）

```ts
/** 對「即將送出的 prompt」掃 customer-facing forbidden terms。命中即代表
 *  deterministicDraft 本身髒（理論上不該發生），回傳命中清單。 */
export function scanRefinePromptLeak(deterministicDraft: string): string[]
```

- 與 output guard 共用 `customer-facing-forbidden-terms.ts`（雙向對稱、單一事實來源）。
- adapter 內 `RefineDraftSource` 在呼叫 LLM 前先掃；命中 → throw `RefinePromptLeakError`（sanitized，不含草稿內容）→ harness 收成 fail-closed。
- 但為了讓 `prompt_leak` 成為**獨立 reason**（修正 2），smoke runner 在呼叫 harness **之前**先用 `scanRefinePromptLeak` 預檢：命中 → 該 case 直接記 `prompt_leak`、**不呼叫 LLM**、不進 harness。adapter 內的 throw 為 defense-in-depth。

### 3.4 source 工廠

```ts
export interface AnthropicRefineDeps {
  apiKey: string
  model?: string                 // 預設 resolveRefineModel()
  /** 可注入；離線單測塞 fake，不碰 SDK。預設動態 import @anthropic-ai/sdk。 */
  callModel?: (req: { system: string; user: string; model: string }) => Promise<string>
}
export function createAnthropicRefineSource(deps: AnthropicRefineDeps): RefineDraftSource
```

接 SDK 時載入 `claude-api` skill（prompt caching 等）。

## 4. CLI loader：`scripts/refine-smoke-runner.mjs`（三閘，修正 1）

仿 `notion-rag-dry-runner.mjs`。三閘：
- `AI_AGENT_REFINE_LLM_ENABLED === 'true'`（feature 閘）
- `AI_AGENT_REFINE_LLM_RUNTIME === 'real'`（真連線閘）
- `ANTHROPIC_API_KEY` present（憑證）

```ts
loadRefineLlmRuntime({ env, importAdapter, createSource })
  → Promise<{ refineSource: RefineDraftSource | null, status, reason }>
```

細分 reason（修正 1）：

| 條件 | status | reason |
|------|--------|--------|
| ENABLED ≠ true | `skipped` | `disabled` |
| ENABLED=true 且 RUNTIME ≠ real | `skipped` | `runtime_not_real` |
| 三閘但缺 key | `client_not_wired` | `missing_key` |
| 三閘但 adapter factory 不可用（plain node 等） | `client_not_wired` | `factory_unavailable` |
| 三閘齊、source 接上 | `real` | `wired` |

- 只有 `real` 才回非 null `refineSource`。
- factory throw → sanitized `RefineLlmWiringError`（不外洩 key）。
- off-gate / 缺 key 在任何 factory 跑之前 short-circuit。

## 6. report：`summarizeRefineSmoke`（純）+ `formatRefineSmokeReport`

```ts
interface RefineSmokeRow {
  caseId: string
  model: string
  status: 'refined' | 'fallback'
  reason: string | null          // fallback 時：lint / structural_diff / forbidden_terms / source_error / empty_output / prompt_leak
  isFallback: boolean
  guardSummary: string           // 簡短 masked 計數，如 'struct=2 leak=0 lint=0'
}
interface RefineSmokeSummary {
  total: number; accepted: number; rejected: number; fallback: number
  acceptRate: number; fallbackRate: number
  byReason: Record<string, number> // lint / structural_diff / forbidden_terms / source_error / empty_output / prompt_leak
  rows: RefineSmokeRow[]
}
```

- accepted = `used==='refined'`；fallback = rejected = `used==='deterministic'`（含 `prompt_leak` 預檢路徑）。
- by-reason 映射：harness `rejectionReasons` 的 `lint_error→lint`、`structural_drift→structural_diff`、`internal_leak→forbidden_terms`、`source_error`、`empty_output`；runner 預檢的 `prompt_leak`。
- 禁印：draft 全文、prompt 全文、token / cost / PII / internal / operator data。per-case 表只為 debug prompt 與 guard。

## 7. 測試（TDD）

- **adapter 單測**（離線注入 fake `callModel`）：
  - `buildRefinePrompt` 只含 system+draft；餵帶 `operatorNotes/retrievalApplications/provenance/themeTag` 字樣的草稿外欄位也進不了 prompt（簽章只收 string）；組好的 prompt 不含這些 token（修正 3）。
  - `resolveRefineModel` 三層覆寫順序。
  - source verbatim 透傳 fake 回傳；fake throw → 傳遞給 harness 成 fail-closed。
  - `scanRefinePromptLeak` 命中髒草稿。
- **loader 單測**：鎖四個 gate case——(1) enabled off 不呼叫 (2) enabled 但 runtime≠real 不呼叫 (3) runtime real 缺 key 不呼叫 (4) 三閘齊才接 source——+ 五種細分 reason + sanitized wiring error。
- **report 單測**：aggregate 數學、by-reason 映射（含 prompt_leak）、masked（無 draft/prompt/cost/PII）。
- **整合**：fixture 草稿 + fake source 經 adapter→harness→report 全鏈，驗 accepted/fallback 分類正確。
- 全程 PURE / fixture-only：零真 LLM、零 RAG live、零 LINE / Sanity / gate。

**真 smoke 不自動跑（修正 4）**：`AI_AGENT_REFINE_LLM_ENABLED=true AI_AGENT_REFINE_LLM_RUNTIME=real ANTHROPIC_API_KEY=… npm run agent:refine-smoke` 一律等 Eric 明確指令手動執行；CC/tmux 不代跑。

## 10. structural_diff 診斷 follow-up（2026-06-09，feature `8d3ef3b`）

**背景**：Eric 已手動跑過一次真 Claude refine smoke（total 4 / accepted 1 / fallback 3，3 個**全部** `structural_diff`，leak 0、lint 0）。但舊 report 的 per-case row 只印 `struct=N` 總數，看不出「3 個 fallback 分別卡在哪個事實」。本刀**只補診斷能力**，讓**下一次** real smoke 能拆出子原因——不重跑真 API、不加 fake drift source、不接 LINE、不翻 gate、不寫 Sanity。

**閉集子原因詞彙**（`StructuralDiffIssue.code` 即子原因 key，純事實「類別」，不含任何值／景點名／日期／金額，故可在 masked report 印出）：
`date_changed` · `people_changed` · `day_count_changed` · `day_order_changed` · `day_date_label_changed` · `day_title_fact_changed` · `activity_line_changed` · `lunch_changed` · `dinner_changed` · `lodging_changed` · `final_day_meal_added`

（沿用舊 11 個 guard 分支，1:1 改名為上述詞彙；單一事實來源＝guard 自身，report 端不另立平行欄位。）

**改動**：
- `customer-itinerary-refine.ts`：guard 11 個 `code` 改名為上述詞彙 + interface JSDoc 記錄閉集與 masking 理由。
- `refine-smoke-runner.mjs`：per-case `guardSummary` 由 `struct=N` 展開為 `struct=N(reasonA=x, reasonB=y)`（依 guard 自然順序、重複次數累計）；`summarizeRefineSmoke` 新增全案 `structuralBreakdown` 彙總；`formatRefineSmokeReport` 加一段「structural 子原因（全案彙總）」masked 行。leak／lint 維持計數不變。
- 範例：`struct=2(activity_line_changed=1, day_title_fact_changed=1) leak=0 lint=0`。

**Masking 不變**：仍不印 deterministic／refined draft 全文、prompt 全文、key／PII／金額／internal／operator data——只多印事實「類別」計數。

**驗證**：TDD RED→GREEN，新增測涵蓋（1）每種 drift 歸正確子原因＋多 drift 各自歸位（2）per-case 展開＋重複累計（3）no-drift 維持 `struct=0`（4）全案彙總數學（5）report 子原因 masked。line-agent **1004/1004** 綠；production `.ts` tsc clean。

**真 smoke 仍不自動跑**：待 Eric 看到 diagnostics landed 後，再由 Eric 明確指令重跑 `npm run agent:refine-smoke`（CC/tmux 不代跑）。

## 11. Day title verbatim freeze follow-up（2026-06-09，feature `a83dec1`）

**背景**：Eric 手動跑的最新 real Claude smoke＝total 4 / accepted 2 / fallback 2，**兩個 fallback 全是 `structural_diff`，子原因全是 `day_title_fact_changed`**（leak 0 / lint 0）。即 LLM 連「Day X｜…」標題裡的景點名都會改寫，而其餘事實都守住了。§10 的診斷讓我們能精準看到這點。

**Eric 決策**：先**不做 fuzzy guard**。把 Day title **完全凍結**，LLM refine v1 只允許潤「最上方標題行＋第一個 Day 之前的開場白＋結尾收尾」這些非 itinerary-fact 表面文字；`Day X｜…` 標題整行需逐字一致。

**改動**：
- `customer-itinerary-refine.ts`：`structuralDiffGuard` 的 Day 標題比對由 `normalizeTitle`（去空白/標點後比對）改為**整行逐字** `da.heading !== db.heading`；移除已成 dead code 的 `normalizeTitle`。`code` 維持 `day_title_fact_changed`（閉集詞彙不變，§10 子原因 report 不破），message 改「Day N 標題行被更動（需逐字一致）」。頂部 doc comment 同步更新（Day-title 不再容忍標點/空白微調）。
- `llm-refine-adapter.ts`：`REFINE_SYSTEM_INSTRUCTION` 明確列「每一個『Day X｜…』標題整行（含標點與空白）必須逐字一字不差」，並把可潤範圍限縮成「最上方 < > 標題行＋開場白／前言＋結尾收尾結語」。

**測試（TDD RED→GREEN）**：
- Day title 只改標點/空白 → 現在 fallback（`day_title_fact_changed`）。
- Day title 改景點名 → fallback（沿用）。
- Day title 完全不動、只動開場/收尾 → 通過（`structuralDiffGuard` 回 `[]`）。
- prompt 斷言含 `Day X｜` 且明列標題逐字凍結。
