## 3. 私人 LINE 測試群前的最低標準（pre private-test-group bar）

1. **隔離目標**：`LINE_PARTNER_GROUP_ID` 指向**私人測試群**，非真夥伴群。確認 group id 來源與值正確（masked 核對，不貼明文）。
2. **送出語意明確**：webhook send gate（`shouldReplyToPartnerGroup`）只在 `line_partner_group` + `botDirected` + decision worthy + 活 replyToken 四條全真才送；確認測試群事件能滿足、且 OA plane 永遠 false（結構性，已驗）。
3. **responder 模式自選**：若只驗 routing/send，維持 `stub`（零 billed）；若要驗真回覆，`anthropic` 但限私人群、單次、人盯。
4. **RAG draft 維持 off**：私測階段 `AI_AGENT_NOTION_RAG_ENABLED` + `AI_AGENT_PARTNER_RAG_DRAFT_ENABLED` 不同時翻，先驗 base responder + send gate + 去重（`claimPartnerReply`）。
5. **失敗不外溢**：確認 reply 失敗被吞（log 不 minified、webhook 仍 200、不 fallback customer/push）——已在 code 內保證，私測需實證一次。
6. **回滾路徑**：一個 env 即可關回（`LINE_ROUTE_MODE=discord_smoke`）；確認回滾後 partner 路徑完全停。

## 4. 正式夥伴群前的最低標準（pre real-partner-group bar）

私測全過後，**額外**要求：

1. **私測群端到端綠**：routing + send-once 去重 + 失敗吞 + 回滾，全部在私人群實證過至少一輪。
2. **RAG draft 端到端**：若要開 partner RAG，`installPartnerRagAnswerSource` 必須真接 Notion source 並在私測群驗過：masked、無 PII/景點名洩漏、fail-closed 命中分支零洩漏、`live_masked` 永不代入。
3. **responder=anthropic 成本閘**：確認 `AI_AGENT_TOOL_COST_CAP_USD` 設定合理；escalation tier（Haiku→Sonnet）在真群的 token 行為已知。
4. **send intent 流程**：貼夥伴群需 Eric 明確 send intent（per CLAUDE.md）；確認沒有任何路徑會在沒有 intent 下自動貼。
5. **customer 隔離複驗**：再次確認 OA customer plane 在 partner_group 模式下仍 100% 不被 auto-reply（結構性，但正式上線前複驗）。
6. **觀測**：log 可讀（非 minified）、可追 claim/send/fail，且不含 PII。

## 6. 風險清單

| 風險 | 嚴重度 | 現況防線 | 殘餘風險 / 行動 |
|------|--------|----------|-----------------|
| **Customer auto-reply** | 高 | 結構性：OA plane botDirected 永遠 false，send gate 只授權 partner | 低；正式上線前複驗一次即可 |
| **Partner group noise** | 中 | send-once `claimPartnerReply` 去重 + 需 botDirected + 活 replyToken + decision worthy | 真群多實例併發未實證；私測需驗去重 |
| **Notion leak** | 高 | masked 投影白名單、`live_masked` 永不代入、命中分支零洩漏、CLI 不印 raw payload | answerSource 真接 Notion 後需重驗 leak（目前未端到端） |
| **LLM drift** | 中 | refine 候選必過三 deterministic guard（lint+structural_diff 逐字+leak），fail-closed 退回 deterministic | 已驗（escalation smoke struct/leak/lint=0）；接 LINE 出文後需再驗 |
| **Sanity write** | 高 | 結構性 dry-run，無 write token | 低；待 Eric 批准 token 才解（另案） |
| **Quote URL** | 中 | slug `DRAFT-<caseId>`、`isOfficial:false`、零 import | 真 `/quote/[slug]` 為未起手階段，需先批 token |

## 7. 建議下一刀（獨立判斷，不迎合）

**建議：先補一刀「composition-level operator preview」，再進私人 LINE 測試群。**

理由：
- 目前所有元件（responder factory / rag surfacing / send gate / refine）都只在**各自隔離**測過；真正進 LINE 的 `defaultEventHandler` 是把它們**組合**起來跑。組合層尚無一次端到端 dry-run。
- 直接跳私人測試群，等於第一次驗組合就同時引入「真送出 + 可能 billed + 真群事件」三個變數，難以歸因。
- 一刀 composition preview（offline，模擬 webhook event → `routeCommand` → outboundText，閘以參數注入 on，**不真送**）可用近乎零成本、零 live 曝險，把組合層先打綠；之後私測群只剩「真送出」單一新變數。

排序：
1. **下一刀**＝composition-level operator preview（offline，不翻 gate、不送）。
2. 其後＝私人 LINE 測試群 smoke（§3 最低標準，stub 先行）。
3. 再後＝partner RAG answerSource 真接 + responder=anthropic（§4）。

> 若 Eric 認為組合層風險可接受、想直接私測群，則跳過步驟 1，但須接受第一次即多變數歸因成本。
