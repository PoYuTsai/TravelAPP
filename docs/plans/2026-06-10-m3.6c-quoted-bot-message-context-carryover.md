# M3.6c — Quoted Bot-Message Context Carryover

- Date: 2026-06-10
- Branch: `codex/line-oa-agent-mvp`
- Scope decision (Eric, 2026-06-10): deterministic customer-safe template +
  fail-closed fallback (NOT LLM-context-only, NOT safe-excerpt of the draft).

## 1. Problem

In the private LINE test group, quoting a bot-authored draft (no re-tag) already
routes to the responder via `deriveBotDirected` → `isBotAuthoredPartnerMsg`. But
the bot-message store tracked **only message ids** (a boolean flag), not the
outbound text. So when a partner quoted a draft and asked「根據我引用的草稿整理
給客人」, the responder had no content and could only answer「請貼上草稿內容」.

## 2. Deliverable

Carry the quoted bot message's **content** into the responder context and, for a
"整理給客人" request, emit a short, conservative, customer-readable suggestion.

### 2.1 Store — cache outbound content (was id-only)
- `CaseStore.putBotAuthoredPartnerMsg(messageId, content?)` — optional `content`;
  when present it is length-capped (`BOT_AUTHORED_CONTENT_MAX_CHARS = 4000`) and
  cached in a sibling namespace. Omitting `content` preserves the prior id-only
  write (backward compatible).
- `CaseStore.getBotAuthoredPartnerMsgContent(messageId): Promise<string | null>`
  — null for empty id (no I/O), id-recorded-without-content, never-recorded, or
  expired. Caller MUST treat null as fail-closed.
- KV namespace `line-agent:partner-bot-msg-content:` (7-day TTL); Memory map.

### 2.2 Deterministic composer (`partner-group/quoted-draft-customer-reply.ts`)
- `QUOTED_DRAFT_CUSTOMER_REPLY` — fixed customer-safe template (Eric-authored):
  suggests a planning direction, makes NO price/availability/feasibility promise,
  reminds to confirm **date / headcount / kids' age / flight / lodging-pickup**.
  It **never echoes the quoted internal draft body** (the draft carries cost /
  margin / internal vocabulary). Cached content's PRESENCE is only the trigger.
- `QUOTED_DRAFT_CONTENT_MISSING_REPLY` — fail-closed paste-the-draft reply when
  content is missing; never guesses, never promises a quote.
- `detectSummarizeForCustomerIntent` — needs an explicit customer mention AND a
  summarise signal (整理/說法/回覆/版本); a bare RAG-lookup probe never matches.
- `sanitizeQuotedBotContext` — whitespace-collapse + trim + cap (1000) for the
  value threaded as context.
- `shouldUseQuotedDraftCustomerReply` — partner group + botDirected + quote event
  (`kind === 'group_quoted'`) + summarise intent. A bare @-tag never qualifies.
- `quotedDraftCustomerResponder` — `PartnerGroupResponder`; template when content
  present, else fail-closed fallback (`degraded`, `error:quoted_draft_content_missing`).

### 2.3 Dispatcher precedence (`responder-factory.ts`)
`createPartnerGroupResponderWithRagDraft` now checks the customer-summary path
**before** rag/base. It is gate-free and never reads Notion; a tagged rag-lookup
(no quote) still takes the rag path, so the existing rag dispatch is not
regressed.

### 2.4 Webhook wiring (`webhook-runtime.ts` → `router.ts` → `handlers.ts`)
- `resolveQuotedBotContent` fetches + sanitizes the quoted content for a
  bot-directed partner-group `group_quoted` event; any store read failure is
  FAIL-SAFE (undefined ⇒ responder fails closed). OA is never bot-directed.
- Threaded as `RouterInput.quotedBotContent` → `handleRespondToPartnerGroup` →
  `PartnerGroupRespondInput.quotedBotContent`.
- On send, the outbound text is cached with the sent id
  (`putBotAuthoredPartnerMsg(id, outboundText)`) so a chained quote carries content.

## 3. Acceptance (TDD)

1. quote bot draft cached + 整理給客人 → customer template; no RAG/Notion/system/
   gate/operator terms; no price/feasibility promise; reminds date/headcount/kids
   age/flight/lodging. ✓
2. quote a human message, no tag → no reply. ✓ (existing quote-to-human test)
3. quote bot but content not cached / fetch throws → fail-closed paste-the-draft. ✓
4. tag bot, no quote (rag-lookup) → existing rag path unchanged. ✓
5. OA customer plane → never auto-replies. ✓ (structural; existing §6.1 regression)

## 4. Hard boundaries (all held)

- No partner production group; private-group test scope only.
- No OA customer auto-reply; no Sanity write; no official quote URL.
- No new gate flipped (the customer path is deterministic + gate-free).
- No internal vocabulary (RAG/Notion/internal/system/gate/operator) in any
  customer-facing output — asserted by `scanCustomerForbiddenTerms`.
- Content missing ⇒ fail-closed (ask to paste), never fabricated.
