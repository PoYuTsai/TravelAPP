## 1. Trigger

Eric smoke-tested the private LINE group (M3.6 Tier A triggers already verified:
no-tag→no-reply, tag→reply, quote-bot→reply, quote-human→no-reply, OA
customer→no auto-reply). A RAG-intent probe in the group:

> 「幫我查內部案例：清邁親子 大象 夜間動物園。請用夥伴內部草稿格式回答，不要編沒有來源的案例。」

returned a **free-form AI** answer ("I can't query the internal case database,
please check Notion/Google Drive yourself") — **not** the deterministic Notion
RAG draft. No PII leaked and nothing was fabricated, but the deterministic RAG /
composer path was clearly never entered.

## 2. Diagnosis (confirmed)

The reply text itself is the diagnostic instrument: the observed answer is a
free-form paragraph, **not** the fixed `PARTNER_RAG_UNAVAILABLE_REPLY`
(「目前內部案例查詢暫時不可用，稍後再試或請 Eric 確認。」). That proves
`shouldUsePartnerRagDraft` returned **false** — a gate-off silent fallback, not a
source-error fail-closed.

`shouldUsePartnerRagDraft` (src/lib/line-agent/partner-group/rag-draft-surfacing.ts)
requires ALL four, AND-joined:

1. `sourceChannel === 'line_partner_group'`
2. `botDirected` (mentionsBot OR quote-to-bot)
3. `detectPartnerRagIntent(text)`
4. `isPartnerRagDraftEnabled(env)` — BOTH gates exactly `"true"`

For the probe, 1–3 all hold (intent token `查內部案例` matches). The chain breaks
at **gate #4**.

### Conclusion (masked)

- **私人群 free-form 的原因 = `AI_AGENT_PARTNER_RAG_DRAFT_ENABLED` disabled**
  (the 2nd gate of the AND).
- **RAG master gate `AI_AGENT_NOTION_RAG_ENABLED` = enabled.**
- **`NOTION_TOKEN` present；`NOTION_PRIVATE_2026_DATABASE_ID` present**
  (`AI_AGENT_NOTION_RAG_ACTIVE_SOURCES = private_2026`) — the source is
  env-wireable; flipping the gate alone would reach the deterministic path.
- intent classifier hit; answer-source wiring is sound; no fabrication.

Live CLI run against the real `.env.local` (masked) reproduced exactly:

```
1. 來源頻道 line_partner_group：✓ PASS
2. botDirected（tag／引用 bot）：✓ PASS
3. RAG intent 命中：✓ PASS
4. RAG 雙閘：
   · AI_AGENT_NOTION_RAG_ENABLED：enabled
   · AI_AGENT_PARTNER_RAG_DRAFT_ENABLED：disabled
5. answerSource 接線（離線 env-presence）：NOTION_TOKEN present；設定 已就緒
6. 最終路徑：fallback free-form responder（非 RAG）
7. 原因：AI_AGENT_PARTNER_RAG_DRAFT_ENABLED 未開（disabled）
```

### Note on Eric's point 5 (fail-closed)

When the gate IS on but the source is not wired, the rag responder correctly
fails closed to `PARTNER_RAG_UNAVAILABLE_REPLY` and never fabricates. The
"pretends it has no permission" free-form refusal only happens when the gate is
**off** — that is the base responder answering honestly, not a wiring bug.

### Note on Eric's point 4 (observability gap, NOT fixed here)

The live private-group path is still **silent** on gate-off (it routes to `base`
with no operator-visible skipped reason). Scope A (this offline tracer) removes
the need to guess from the real group. Adding a live operator-log skipped reason
would be a separate cut (the original "option C", touches responder runtime) and
was deliberately deferred.

## 3. Deliverable — offline path tracer (scope A)

- `src/lib/line-agent/partner-group/rag-path-trace.ts`
  - `tracePartnerRagPath(input) → PartnerRagPathTrace` — pure; reuses the SAME
    runtime decision functions so it cannot drift.
  - `formatPartnerRagPathTrace(trace, text) → string` — masked TC report.
  - Offline wiring check reuses `resolveNotionRagConfig` with a forced-enabled
    overlay on a COPY of env (assesses present/missing independently of the
    master gate) — no live Notion read, no real gate flip.
- CLI subcommand `partner-rag-path-trace` in `scripts/agent-command.mjs`
  (guard-loaded dynamic import of the pure TS kit; injectable `traceKit` for tests).
- npm script `agent:partner-rag-path-trace`:
  ```
  npm run agent:partner-rag-path-trace -- "幫我查內部案例：清邁親子 大象 夜間動物園"
  ```

## 4. Hard boundaries (all held)

- 不接 LINE（no send, no webhook path)
- 不讀 Notion live（env-presence only; imports no Notion client)
- 不跑 LLM
- 不寫 Sanity
- 不翻 gate（forced-enabled overlay is a pure-resolver arg on an env copy)
- masked output — only enabled/disabled/present/missing/PASS/FAIL; never an env
  value, token, db id, or Notion url.
